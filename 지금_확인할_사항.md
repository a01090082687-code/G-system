# 지금 확인할 사항 체크리스트

## ✅ 1단계: 브라우저 콘솔 확인 (가장 중요!)

현재 개발자 도구가 열려있으니:

1. **콘솔 탭 클릭** (Sources 옆에 있는 "Console" 탭)
2. 다음 메시지들을 확인하세요:

### 정상 작동 시 보이는 메시지:
```
📡 Google Sheets 데이터 로드 시도: https://script.google.com/...
✅ Google Sheets 데이터 로드 성공: {members: [...], risks: [...], ...}
```

### 문제가 있을 때 보이는 메시지:
```
❌ Google Sheets 데이터 로드 실패: Error: ...
⚠️ GAS_URL이 비어 있어 저장되지 않습니다.
```

---

## ✅ 2단계: Google Sheets 확인

1. Google 스프레드시트를 열어서 확인:
   - 시트 탭이 6개 있는지 확인: `Members`, `Risks`, `Noise`, `Expenses`, `Innovations`, `Config`
   - 각 시트의 첫 번째 행(헤더)이 올바른지 확인
   - 데이터가 붙여넣어졌는지 확인

---

## ✅ 3단계: Google Apps Script 배포 확인

1. 스프레드시트에서 **확장 프로그램** → **Apps Script** 클릭
2. **배포** → **웹 앱 배포 관리** 클릭
3. 배포가 되어 있는지 확인
4. **액세스 권한**이 **"모든 사용자"**인지 확인

---

## ✅ 4단계: index.html의 GAS_URL 확인

1. `index.html` 파일 열기
2. `GAS_URL` 검색 (Ctrl+F)
3. URL이 올바르게 설정되어 있는지 확인:
   ```javascript
   const GAS_URL = 'https://script.google.com/macros/s/.../exec';
   ```
4. 빈 문자열(`''`)이 아닌지 확인

---

## ✅ 5단계: 관리자 모드에서 저장 테스트

1. 브라우저에서 관리자 모드로 로그인 (비밀번호: 8078)
2. 아무 데이터나 수정 (예: 유해인자 추가, 혁신 성과 추가 등)
3. 콘솔에서 다음 메시지 확인:
   ```
   💾 Google Sheets에 저장 시도: saveRisks [...]
   ✅ Google Sheets 저장 성공: saveRisks
   ```
4. Google Sheets를 새로고침하여 데이터가 저장되었는지 확인

---

## 문제 해결

### 콘솔에 에러가 보이는 경우:

**"GAS_URL이 비어 있어 저장되지 않습니다"**
→ `index.html`의 `GAS_URL`을 확인하고 올바른 URL로 설정

**"Google Sheets 데이터 로드 실패"**
→ Google Apps Script가 "모든 사용자"로 배포되었는지 확인

**"HTTP 401" 또는 "HTTP 403"**
→ Google Apps Script 권한 문제 → "모든 사용자"로 재배포 필요

**"HTTP 404"**
→ GAS_URL이 잘못됨 → 새로 배포한 URL로 업데이트
